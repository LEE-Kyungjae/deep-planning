## Launch Post

Palamedes is a local planning engine for builders working with AI.

It is built for the layer before execution:
- what to build
- why now
- what evidence supports it
- what would invalidate it
- when to replan

Palamedes is intentionally plan-only. It does not run tasks, schedule workflows, or own delivery.

Instead, it keeps planning state in your repo with:
- one current plan
- explicit evidence and hypotheses
- revision history
- restore points
- a defined moment to replan

If AI is helping you build fast but direction keeps drifting, this is the problem Palamedes is meant to solve.

## Blog Intro

AI reduced the cost of execution, but it did not reduce the cost of choosing the wrong direction. Palamedes is an attempt to make planning state explicit: one current plan, the evidence behind it, the signals that would invalidate it, and a revisioned history of how that direction changed over time.

## Conference Talk Intro

Most teams now have more ways to generate output. What they still lack is a durable way to decide what matters before more output accumulates. Palamedes is a local, versioned planning state designed for that gap.
